# Ennodia

This repo is Bun-first: use `bun install` for dependencies and
`bun run verify` before handing off changes. Project rules — thin adapters,
no permission-bypass flags, no provider-private APIs — are in
[CONTRIBUTING.md](CONTRIBUTING.md); follow them here too.

Keep progress updates compact and factual. Prefer short bullets; do not
repeat full task details on every poll/tick unless the state changed. A good
update is like: `* Working tree is clean.` or `* Harnesses are available:
Gemini -> Antigravity; Kimi/Minimax/GLM -> OpenCode; Opus -> Claude Code;
GPT -> Codex.`
